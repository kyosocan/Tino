# Tino 产品需求文档（PRD）

> 本文档依据当前代码库实现整理，用于对齐「已有能力」与后续迭代方向。  
> 技术栈：Next.js 15、React 19、Tailwind CSS；服务端状态为进程内内存（重启即清空）。

---

## 1. 产品概述

### 1.1 产品定位

**Tino** 是一款面向低龄学习者的 **英语陪练与轻社交** Web 应用：以拟人角色「小狐狸 Tino」为核心，提供 **1v1 AI 陪伴对话**、**随机匹配英语聊天室**（真人或 AI 替补）、**漂流瓶** 等玩法，并通过 **钻石与头像框** 做轻量激励。

### 1.2 目标用户

- 小学段儿童及其家长场景（界面文案、交互偏「大按钮、少步骤、强反馈」）。
- 需在浏览器中使用麦克风（语音输入 / 部分场景语音附件）。

### 1.3 产品目标（从实现反推）

- 鼓励用户 **多说英文**：通过对话、房间互动与钻石计分强化输出。
- 降低开口门槛：中文输入时提供 **翻译 + 跟读引导（TTS）**，同音词（如「嗨」→ Hi）直接视为英文。
- **冷启动可玩**：匹配久等时自动进入 AI 同伴房间；漂流瓶海中有 **虚拟小朋友瓶子** 池，保证可捞。

## 2. 信息架构与模式

主界面以 **手机外框（Device Frame）** 呈现，应用内模式（`AppMode`）包括：

| 模式 | 说明 |
|------|------|
| `companion` | 默认首页：与 Tino 一对一聊天（主流程） |
| `matching` | 匹配中（动画与排队提示） |
| `room` | 英语聊天室（真人房间或 AI 同伴房间） |
| `shop` | 头像框商店（钻石消费） |
| `bottle` | 漂流瓶（扔 / 捞 / 回复 / 收件箱） |

类型定义中含 `login`，**当前主页面未使用该模式**（用户身份以本地存储 + 会话级 `userId` 为主）。

## 3. 用户与身份

### 3.1 用户标识

- **`userId`**：优先 `sessionStorage.tino_user_id`，无则生成 UUID 并写入。
- **展示名与年级**：初始化示例为固定「小明」、年级 `1`（可按产品需求改为可编辑资料页）。

### 3.2 持久化（浏览器）

| Key | 用途 |
|-----|------|
| `tino_diamonds` | 钻石余额 |
| `tino_owned_frames` / `tino_active_frame` | 已购与当前头像框 |
| `tino_friends_list` | 好友列表（名称、年级、最近聊天时间） |
| `tino_companion_memory_*` | 陪伴记忆（偏好句、轮次统计等） |
| `tino_user_name` / `tino_user_grade` | 用户名与年级 |

## 4. 功能说明（已实现）

### 4.1 设备外框与基础控制

- **电源**：控制设备「开机」状态；与问候 TTS 播放等联动。
- **音量**：0–10 档，影响 TTS 播放增益；短时显示音量条。
- **音频解锁**：用户手势内初始化 `AudioContext`，并尝试请求 **设备运动权限**（用于摇一摇等）。

### 4.2 陪伴模式（Companion）— 与 Tino 一对一

**对话**

- 调用 **`POST /api/chat`**：基于豆包流式回复（`text/plain` 流）；失败时回退 JSON 兜底文案。
- 系统提示词约束：中英自然穿插、短句、鼓励式、不做语法课与打分。
- **轮次与引导**：`turnCount` 每 2 轮注入一次「英文引导」系统提示。
- **长期记忆注入**：根据本地 `CompanionMemory` 传入 `userMemory`（偏好记忆、薄弱点摘要、总轮次与英文轮次等）。

**流式展示与朗读**

- 流式内容按句切分后 **逐句请求 TTS**（`/api/tts`），字幕区与当前句同步。
- 非流式响应则整段切句后朗读。

**记忆与薄弱点（客户端逻辑）**

- 从用户消息中抽取「你喜欢××」类偏好，最多保留若干条。
- 根据英文占比、平均词长、提问比例等生成 **薄弱点备注**，供模型参考。

**钻石激励**

- 用户发送内容时按 **英文复杂度启发式打分**（`scoreEnglish`），增加钻石；陪伴模式直接计入总钻石。

**首次进入引导（Onboarding）**

- 定时推送多条 Tino 介绍与例句；第二步后出现 **阅读卡**，引导跟读「Nice to meet you!」；完成后引导 **摇一摇匹配**。

**语音输入**

- 按住说话 → **`POST /api/asr`** 转写 → 将文本送入对话（陪伴与房间逻辑共用能力）。

**说明**：`components/CompanionChat.tsx` 为独立示例实现（非流式、无记忆），**主站实际逻辑集中在 `app/page.tsx`**。

### 4.3 匹配与英语聊天室（Room）

**进入方式**

- **显式入口**：如「开始匹配」类操作进入 `matching`。
- **摇一摇**：在陪伴模式下检测 `devicemotion`，在阈值与时间窗口内触发 `startMatch`（有冷却）。

**匹配服务** — **`POST /api/match`**

| action | 行为 |
|--------|------|
| `join` | 入队；若队列中有他人则立即成房并返回 `roomId`、对方信息 |
| `status` | 轮询是否已匹配 |
| `leave` | 离开队列并离开房间映射 |

**队列规则（服务端 `matchStore`）**

- 等待队列中单用户 **60s** 未匹配则剔除（防 stale）。
- 成房时插入 **Tino 欢迎消息**（英文为主）。

**真人房间** — **`POST /api/room-chat`**

| action | 行为 |
|--------|------|
| `send` | 发送文本；可选 `audioBase64` + `mimeType`（语音消息） |
| `poll` | 按 `since` 时间戳增量拉取消息 |
| `info` | 房间元信息 |

- 每 **4 条非 Tino 用户侧消息**（实现上为用户消息计数模 4）触发 **Tino 插话**：异步调用 LLM 生成短评论并写入房间（`shouldTinoComment` / `addTinoMessage`）。

**AI 同伴房间（替补）**

- 若 **`join` 后约 10s 仍未匹配到真人**：离开队列并进入 **AI 同伴房间**（`isAiRoom`）。
- **`POST /api/ai-partner`**：模拟同龄英文对话，1–2 句英文为主、少量中文辅助。

**从好友列表「拨打」**

- 可选择好友进入 **呼叫动画**，约 3s 后以该好友姓名为名进入 **AI 房间**（仍为 AI 对话，非真实呼叫）。

**房间时长**

- 单次 **300 秒** 倒计时；结束系统提示并回到陪伴模式（过程中合并本会话钻石到总余额）。

**房间内中文处理**

- 若用户发送含中文且不属于「同音英文」白名单：调用 **`POST /api/translate`**（中→英 JSON：整句、分词音标、voiceGuide），再由 Tino **分句 TTS** 引导跟读，不直接当英文消息发出。

**会后小结（Debrief）**

- 离开房间或定时结束时，后台请求 **`POST /api/debrief`**，生成 **两段中文反馈**（禁止「上课感」用词），返回主界面后择机展示。

**其他 API**

- `POST /api/room`：独立房间评论/总结（`tino_comment`、`summary` 等），供 **`components/RoomView.tsx`** 等演示组件使用；**主流程房间以 `room-chat` + 页面内 UI 为准**。

### 4.4 漂流瓶（Drift Bottle）

**接口** — **`POST /api/drift-bottle`**

| action | 行为 |
|--------|------|
| `throw` | 扔瓶：文本必填，可选语音附件 |
| `pick` | 捞瓶：不能捞自己的；服务端维护池子 |
| `reply` | 对捞到的瓶子回复一次 |
| `inbox` | 查看自己扔出的瓶子收到的回复（按回复时间排序，最多 20 条） |
| `count` | 当前可捞数量（含虚拟瓶池维护） |

**服务端逻辑（`bottleStore`）**

- 全局最多保留 **500** 个瓶子；**虚拟小朋友瓶子** 在未捞数量不足时自动补足，保证冷启动可捞。
- 虚拟瓶有固定英文名池与英文模板句。

**前端子状态**

- 菜单：扔瓶 / 捞瓶 / 收件箱；扔瓶输入、动画、回复、收件箱列表等。

### 4.5 商店（头像框）

- 多档 **头像框**（默认免费 + 多款付费），价格以钻石标价。
- 购买：扣减钻石并解锁；可切换「使用中」。
- 头像框样式作用于界面中的头像展示。

### 4.6 好友列表

- 与真人匹配成功后 **写入/更新** 好友记录（本地最多约 20 条）。
- 用于展示、未读样式提示（如与漂流瓶收件配合的 HUD），以及「拨打」进入 AI 房间。

### 4.7 翻译与语音能力

| 接口 | 作用 |
|------|------|
| `POST /api/translate` | 中→英（JSON：句子、单词音标、朗读引导语）；英→中（`lang: en2zh`） |
| `POST /api/tts` | 文本转语音（返回 base64 音频） |
| `POST /api/asr` | 语音识别 |

底层对接火山方舟等配置见 `scripts/ai_api/` 与环境变量（如 `ARK_*`）。

## 5. 非功能与约束

### 5.1 数据与多实例

- **匹配房间、漂流瓶** 等状态存于 **Node 进程全局变量**，无跨进程/跨机一致性；**生产多实例或 Serverless 需改为 Redis 等共享存储**。

### 5.2 隐私与安全

- 语音与对话经第三方 ASR/LLM/TTS；需在隐私政策中说明。
- 无账号体系时，`userId` 仅会话级，换浏览器即新身份。

### 5.3 依赖环境

- 需配置可用的方舟/豆包等 API Key 与模型名，否则对话、翻译、语音能力会失败或走兜底文案。
