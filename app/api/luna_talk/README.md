# Luna Talk 英语对话系统设计

## 概述

Luna Talk 是一个灵活的英语对话学习系统，通过模块化的 Prompt 管理实现个性化的英语学习体验。

## 目录结构

```
luna_talk/
├── README.md                    # 本设计文档
├── route.ts                     # API 路由入口
└── data/                        # Prompt 数据文件夹
    ├── 人设/                    # AI 角色设定
    │   └── luna.md
    ├── 模式/                    # 对话模式
    │   ├── 中英混合.md
    │   └── 纯英文.md
    ├── 教学/                    # 教学目标
    │   └── 基础词汇/
    │       └── easy.md
    ├── 故事/                    # AI 背景故事
    │   └── 我的学校.md
    ├── 技能/                    # 聊天技能/游戏
    │   └── 词语接龙.md
    └── 记忆/                    # 用户记忆
        ├── 用户画像.md
        └── 习得词汇.md
```

## Prompt 组成结构

每次对话的系统提示词由以下 5 部分组成：

| 部分 | 文件夹 | 说明 | 是否必需 |
|------|--------|------|---------|
| 1. 人设 | `人设/` | AI 的性格、名字、说话风格 | ✅ 必需 |
| 2. 对话模式 | `模式/` | 中英混合、纯英文等风格选择 | ✅ 必需 |
| 3. 练习目标 | `教学/` | 词汇、句式、考试目标等 | ⭕ 可选 |
| 4. 话题/技能 | `故事/` + `技能/` | 聊天话题背景、互动游戏 | ⭕ 可选 |
| 5. 用户记忆 | `记忆/` | 用户画像、已习得内容、关键事件 | ⭕ 可选 |

## MD 文件格式规范

所有 data 文件夹下的 Markdown 文件必须遵循以下格式：

```markdown
---
name: 名称
description: 简短描述（用于选择逻辑判断）
---

这里是文件的具体内容...
```

### Frontmatter 字段说明

| 字段 | 类型 | 说明 | 是否必需 |
|------|------|------|---------|
| `name` | string | 配置项的名称 | ✅ 必需 |
| `description` | string | 配置项的描述，用于智能选择 | ✅ 必需 |

### 内容部分

Frontmatter 之后是自由的 Markdown 内容，可以包含：
- 标题、段落、列表
- 示例对话
- 规则说明
- 任何需要的文本内容

## 记忆管理

`记忆/` 文件夹特殊说明：
- 这是**可写**的文件夹（其他只读）
- 随着对话进行会更新内容
- 持久化用户的学习进度和个人信息

### 记忆文件列表

| 文件 | 内容 | 更新频率 |
|------|------|---------|
| 用户画像.md | 用户基本信息、学习背景 | 初次使用时 |
| 习得词汇.md | 用户已掌握的单词表 | 每次对话后 |
| 关键事件.md | 重要的对话里程碑 | 达到目标时 |

## 使用示例

### 基础使用（默认配置）

```
系统提示词 = 人设/luna.md + 模式/中英混合.md
```

### 完整配置

```
系统提示词 =
  人设/luna.md +

  故事/我的学校.md +
  记忆/用户画像.md +
  记忆/习得词汇.md
```

## 扩展指南

### 添加新的人设

1. 在 `人设/` 文件夹下创建新的 `.md` 文件
2. 遵循格式规范，填写 `name` 和 `description`
3. 编写角色设定内容

### 添加新的对话模式

1. 在 `模式/` 文件夹下创建新的 `.md` 文件
2. 描述该模式的特点和适用人群
3. 给出示例对话

### 添加新的教学目标

1. 在 `教学/` 下创建子文件夹（如 `中级句式/`）
2. 按难度创建文件（`easy.md`, `medium.md`, `hard.md`）
3. 描述教学目标和重点内容

### 添加新的技能/游戏

1. 在 `技能/` 文件夹下创建新的 `.md` 文件
2. 描述游戏规则
3. 给出游戏示例
